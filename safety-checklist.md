# 安全检查清单与问题处理预案

## 前期检查（动手前必须全部完成）

- [ ] 工作区干净：`git status` 无未提交改动，建议从稳定基线拉出 `refactor/xxx` 分支
- [ ] 已确认编译/构建命令并成功跑通（见 verification-commands.md）
- [ ] 已确认下游依赖模块（改公共类时需一并编译）
- [ ] 已生成《重构清单》（含业务域分类表）
- [ ] 已在项目根创建 `REFACTOR_PROGRESS.md` 并填入基线 commit
- [ ] 已整理对外接口基线清单
- [ ] 已评估测试现状；核心类无测试的，先补特征测试
- [ ] 已识别高危区并标注「默认不碰」
- [ ] **已用 AskQuestion 获得用户「可以开始」确认**（未确认禁止改业务代码）

## 中期检查（每个重构单元）

- [ ] 本单元只做一种重构（单域内不跳批）
- [ ] 删除/移动前已全局搜索引用（Java + XML + yml + 反射字符串）
- [ ] 编译通过
- [ ] 相关测试通过
- [ ] 对外接口签名与基线一致
- [ ] **批次门禁已通过**（verification-commands.md）
- [ ] 批次 8：方法重命名已查 XML/反射；URL 路径未变
- [ ] `REFACTOR_PROGRESS.md` 已更新（含门禁记录）
- [ ] 已给出 commit 建议 message；用户要求时代为 commit
- [ ] 新发现的 bug / 死代码已登记，未顺手修改

## 后期检查（收尾）

- [ ] 全量构建通过（含下游模块）
- [ ] 全量测试通过
- [ ] 接口清单与基线 diff 为空（或差异已获用户确认）
- [ ] 已输出《规范符合度报告》（templates/refactor-report.md）
- [ ] 全项目门禁扫描已执行并附在报告中
- [ ] 死代码清单已逐项问用户并处理
- [ ] README / 接口文档已更新
- [ ] 规范已沉淀为 `.cursor/rules/`
- [ ] 新坑已回写 smells-playbook.md

## 问题处理预案

总原则：**5 分钟内修不好就回滚该单元，缩小步子重做。**

| 问题 | 处理方式 |
|------|---------|
| 编译失败 | 立即修复；修不好则回滚，拆更小单元 |
| 测试失败 | 判断：测试固化 bug（更新测试并登记）还是重构改坏（回滚） |
| 接口出入参变化 | 红线违规，回滚 |
| ClassNotFound / Bean 找不到 | 查组件扫描、XML 全限定名、配置类名字符串 |
| 循环依赖 | 引入第三类承载共享逻辑或重划边界；禁止 `@Lazy` 掩盖 |
| 配置外置后启动失败 | 查漏改的 profile；配置类加 `@Validated` |
| 删死代码后有隐藏引用 | `git revert`；补充到前期检查清单 |
| 分库分表 SQL 异常 | 回滚；Mapper 列入高危区 |
| 分支冲突 | 每 1-2 天 rebase；大单元延后 |
| 上下文丢失 | 靠 `REFACTOR_PROGRESS.md` + 每单元边界记录恢复 |
| **门禁未通过却想进下一批** | 禁止；继续当前批直至通过 |
